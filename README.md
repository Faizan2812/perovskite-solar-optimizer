# Perovskite Solar Cell Design & Optimization Tool

> An open-source, AI-driven replica of SCAPS-1D's core workflow for 1D
> perovskite solar cells — extended with a real physics-informed neural
> network, machine-learning surrogates, inverse design, and a material
> database with per-parameter provenance.

**Quick demo**: `streamlit run app.py`
**Full benchmark**: `python run_final_benchmark.py`
**Tests**: `pytest tests/`

---

## Why this tool exists

SCAPS-1D is the community workhorse for perovskite simulation, but it is:
- Windows-only, GUI-driven, one cell at a time
- Closed source — you can't audit its numerics
- Has no built-in optimizer, and no way to verify where material parameters came from

Published perovskite ML and PINN studies routinely cite untracked material
parameters ("taken from prior simulation work") that can't be traced to an
experimental measurement. For a tool to be usable in publications, every
number it uses needs a DOI behind it and a confidence tier you can audit.

This project rebuilds that workflow as an auditable, scriptable, validated
Python package.

## What's inside

| Component | What it does | File(s) |
|---|---|---|
| **Drift-Diffusion solver** | Scharfetter-Gummel PDE solver, validated to 9.1% median PCE error vs 10 published SCAPS references | `physics/dd_solver.py` (875 lines) |
| **Fast analytical simulator** | Single-diode surrogate with Tauc absorption; sub-50 ms per J-V sweep | `physics/device.py` |
| **Real PINN** | PyTorch PINN with Poisson + continuity residuals via autograd; log-space carriers; Fourier features | `ai/pinn_real.py` (~400 lines) |
| **ML surrogates** | 7 regressors (RF, XGB, CatB, GBoost, SVR, KNN, BRidge) + MLP | `ai/ml_models.py` (629 lines) |
| **Evolutionary optimizers** | GA, DE, NSGA-II, Bayesian optimization, inverse design | `ai/optimizer.py` (1076 lines) |
| **Material database** | 20 materials · 179 parameters · 29 verified references · DOI per value | `data/materials_database.json` |
| **Experimental benchmarks** | 5 fabricated, measured devices from peer-reviewed papers | `data/experimental_benchmarks.json` |
| **Streamlit UI** | 10 tabs covering simulation, ML, PINN, optimization, database | `app.py` |

## Key feature: per-parameter provenance

Every material parameter in this tool carries:

- **Value** (e.g., `Eg = 1.55 eV`)
- **Source** (e.g., `Green et al., Nat. Photon. 8, 506 (2014)`)
- **DOI** (e.g., `10.1038/nphoton.2014.134`)
- **Measurement method** (e.g., `photoluminescence spectroscopy`)
- **Confidence tier** (`HIGH` / `MEDIUM` / `LOW`)

Open the **📋 Database tab** in the app to inspect any material. Every value
you see in a simulation has a verifiable origin.

**Provenance breakdown of the shipped database**:
- HIGH confidence: 25 parameters (experimentally verified, widely cited)
- MEDIUM confidence: 147 parameters (literature consensus values)
- LOW confidence: 7 parameters (flagged for re-sourcing before publication)

The LOW-confidence list is printed by `summarize_provenance_confidence()` and
by CI on every commit. Don't publish PCE numbers derived from LOW-confidence
inputs without re-sourcing them.

## Validation

Two independent benchmark suites run on every commit:

**1. SCAPS-reference benchmark** (10 published simulation results)
- Mean PCE error: 9.1% (median 8.7%)
- Worst case: 17% (Cs₂SnI₆ device with asymmetric mobilities)
- All 10 converge

**2. Experimental benchmark** (5 fabricated, measured devices)
- Saliba 2016 (EES 9, 1989) — certified 21.1%
- Jeon 2018 (Nat. Energy 3, 682) — certified 22.6%
- Liu 2013 (Nature 501, 395) — 15.4%
- Wang 2019 (ACS EL 4, 222) — lead-free 6.36%
- Kim 2019 (Joule 3, 2179) — certified 23.7%

See `docs/VALIDATION.md` for full interpretation guide and `BENCHMARK_REPORT.md`
(generated by `scripts/run_benchmark.py`) for the latest numbers.

## Common workflows

### Run a single simulation

```python
from physics.device import simulate_iv_curve
from physics.materials import HTL_DB, PEROVSKITE_DB, ETL_DB

result = simulate_iv_curve(
    HTL_DB["Spiro-OMeTAD"], PEROVSKITE_DB["MAPbI3"], ETL_DB["TiO2"],
    d_htl=200, d_abs=500, d_etl=50,
    Nt=1e14, temperature=300,
    mode="dd",
)
print(f"PCE = {result['PCE']:.2f}%")
```

### Train a PINN

```bash
python scripts/train_pinn.py --device D1 --epochs-data 1000 --epochs-pde 5000
```

### Inverse design

```bash
python scripts/run_inverse.py --target-pce 24 --target-voc 1.15
```

### Regenerate all benchmarks

```bash
python run_final_benchmark.py
# Produces: BENCHMARK_REPORT.md, EXPERIMENTAL_BENCHMARK_REPORT.md
```

### Add a new material

```bash
# 1. Edit data/materials_database_extension.json
# 2. Merge into the main DB
python scripts/merge_materials_db.py
# 3. Run tests
pytest tests/
# 4. Confirm benchmarks still pass
python scripts/run_benchmark.py
```

See `docs/MATERIAL_LIFECYCLE.md` for the full workflow.

## Scope and limits

**What this tool handles well**
- 1D perovskite device simulation (single-junction)
- Design-space exploration across layer thickness, doping, defect density
- Comparison against published SCAPS references
- Material parameter uncertainty tracking

**What it does NOT handle (yet)**
- 2D/3D effects: grain boundaries, pinholes, lateral inhomogeneity
- Ion migration, hysteresis, and long-term degradation
- Tandem cells with real tunnel-junction physics (TandemConfig is a stub)
- Excitonic effects in 2D/low-dimensional perovskites

If your work depends on any of these, this tool will mislead you. Use it
for the regime it was designed for: 1D thin-film PSC device modelling in
the classical DD framework.
